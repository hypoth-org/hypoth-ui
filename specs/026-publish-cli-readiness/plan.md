Prepare all 12 publishable design system packages for npm release under the `@hypoth-ui/*` scope by: (1) renaming packages from `@hypoth-ui/*` to `@hypoth-ui/*`, (2) adding required metadata to all package.json files, (3) expanding the CLI's copy-mode with a `--copy` flag and auto-discovery template syncing, (4) writing comprehensive README documentation with tiered package listing and dual getting-started guides, and (5) validating the release pipeline end-to-end.

## Implementation Phases

### Phase 1: Package Rename (@hypoth-ui/* → @hypoth-ui/*)

The largest and most impactful change. Must be done first as all subsequent work depends on correct package names.

**Steps**:
1. Update all 16 `package.json` `name` fields
2. Update all `dependencies`, `devDependencies`, `peerDependencies` references across all package.json files
3. Batch find+replace `from '@hypoth-ui/` → `from '@hypoth-ui/` across 163+ source files
4. Update `import '@hypoth-ui/` → `import '@hypoth-ui/` (side-effect imports)
5. Update `.changeset/config.json` fixed pattern
6. Update root `package.json` script filter references
7. Update CLI template transform rules (sync-templates.ts references `@hypoth-ui/`)
8. Run `pnpm install` to regenerate lockfile
9. Run `pnpm build && pnpm test && pnpm typecheck && pnpm lint` to validate

**Risk**: High — touches every package. Mitigated by comprehensive test suite validation after rename.

### Phase 2: Package Metadata

Add required publishing metadata to all 12 publishable packages.

**Steps**:
1. Add `license`, `repository`, `keywords`, `description`, `homepage` to each publishable package.json
2. Use `@hypoth-ui/cli` as reference template for field format
3. Ensure `description` includes "Alpha" caveat or note
4. Validate all fields with the contract schema

### Phase 3: CLI Copy-Mode Enhancements

Add `--copy` flag and auto-discovery template syncing.

**Steps**:
1. Add `-c, --copy` option to CLI `add` command definition (`packages/cli/src/index.ts`)
2. Update `AddOptions` type to include `copy?: boolean`
3. Update `add.ts` to check `options.copy` before `config.style`
4. Replace hardcoded component list in `sync-templates.ts` with auto-discovery:
   - Scan `packages/react/src/components/` and `packages/wc/src/components/`
   - Filter against registry entries
   - Log skipped/missing components
5. Run `pnpm sync:templates` to populate all templates
6. Update registry `components.json` to reflect actual template availability
7. Add error message for `--copy` on components without templates
8. Build and test CLI

### Phase 4: Documentation

Write README files and getting-started guides.

**Steps**:
1. Write repository `README.md`:
   - Hero section with name, one-liner, Alpha badge
   - Core Packages table (react, wc, tokens, css, next, cli)
   - Getting Started — Package Mode
   - Getting Started — Copy Mode
   - Package vs Copy comparison table
   - Framework quick-starts (React, WC, Next.js)
   - Tooling & Documentation packages table
   - Contributing, License
2. Write package-level `README.md` for each of the 12 publishable packages
3. Each package README: Alpha badge, install command, basic usage, link to main docs

### Phase 5: Release Pipeline & First Changeset

Validate and enhance the release workflow, create initial changeset.

**Steps**:
1. Add NPM_TOKEN validation step to `.github/workflows/release.yml` (before build)
2. Create initial changeset (minor bump: 0.0.0 → 0.1.0, all packages)
3. Trigger dry-run release to validate end-to-end
4. Document release process in README or CONTRIBUTING.md
